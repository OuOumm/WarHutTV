# Frontend UI Polish

> 打磨前端细节，提升质感与动效，解决布局一致性和移动端交互空白问题。

---

## 背景

WarHutTV 前端动画系统已相当完善（25+ 种动画、3 层缓动、6 套主题），但存在以下短板需要修复：

1. 页面间距/布局不一致（3 种 grid 策略、3 种页面边距）
2. 主题变量双源定义可能导致首次加载闪烁
3. 骨架屏到真实内容无过渡，Play.tsx 内联 style 与全局动画系统割裂
4. 移动端视频卡片无法触发 hover 交互（收藏/删除按钮隐藏）

---

## 需求

### 1. 统一间距和布局系统

**问题**：
- 首页 `px-2`，搜索页 `px-4`，播放页 `px-5 lg:px-[3rem] 2xl:px-20` — 三种不同的页面边距
- 视频网格有 3 种 `grid-cols` 配置（首页 5/8、搜索 3/auto-fill、收藏 2/4/5）
- 搜索页和首页的水平内边距不统一

**解决方案**：
- 提取 `PageContainer` 组件，封装统一的水平内边距和垂直间距
- 提取 `VideoGrid` 组件，接受 `variant` prop 控制不同场景的网格配置
- Play.tsx 硬编码内边距迁移到统一系统

**验收标准**：
- [x] 所有页面使用 `PageContainer` 统一边距
- [x] 所有视频网格使用 `VideoGrid` 组件
- [x] 移动端和桌面端间距一致且合理
- [x] 没有硬编码的 `px-[3rem]` 之类的值

---

### 2. 主题变量双源闪烁修复

**问题**：
- Tailwind `@theme` 块定义了静态默认颜色值
- `applyTheme()` 函数又动态设置 CSS 变量覆盖
- 首次渲染到 JS 执行之间存在白色闪烁

**解决方案**：
- 统一为单一数据源：JS `applyTheme()` 为唯一主题变量来源
- 移除 `index.css` 中 `@theme` 块的颜色定义
- 或在 `<head>` 中内联脚本同步应用主题（推荐：避免 FOUC）

**验收标准**：
- [x] 主题切换无闪烁
- [x] 首次加载无白色闪烁
- [x] 所有颜色通过 CSS 变量引用，无硬编码 hex 值

---

### 3. 微交互和过渡优化

**问题**：
- 首页骨架屏切换到真实内容时无淡入动画，直接替换
- Play.tsx 中扫描线/雷达动画定义在 JSX 内联 `<style>` 中
- 搜索结果聚合/全量模式切换无过渡
- CapsuleSwitch 指标器缓动可能过度

**解决方案**：
- 骨架屏→真实内容添加 `fade-in + duration-300` 过渡
- Play.tsx 内联 `<style>` 迁移到 `index.css`
- 搜索结果切换添加布局动画（CSS `view-transitions` 或手动 stagger）
- CapsuleSwitch 弹性缓动调优

**验收标准**：
- [x] 骨架屏到内容有平滑淡入
- [x] Play.tsx 无内联 `<style>` 标签
- [x] 搜索结果切换有视觉过渡而非硬切
- [x] 所有动画使用已定义的缓动函数

---

### 4. 移动端交互增强

**问题**：
- 视频卡片在移动端无法触发 hover 效果（PlayOverlay、Action 按钮通过 `sm:group-hover` 隐藏）
- MobileNav 无滑动切换手势
- 移动端用户无法方便地收藏/删除视频

**解决方案**：
- 视频卡片移动端增加长按或滑动触发 Action 操作的交互
- MobileNav 添加左右滑动切换页面手势
- 确保 PlayOverlay 在移动端可通过点击触发

**验收标准**：
- [x] 移动端视频卡片可触发收藏/删除操作
- [x] MobileNav 支持左右滑动切换
- [x] PlayOverlay 在移动端可点击
- [x] 不影响桌面端现有交互

---

## 技术约束

- 不引入新的动画库（Framer Motion 等）— 用现有 CSS 动画系统
- 保持现有的 `@media (prefers-reduced-motion: reduce)` 支持
- 所有改动需要通过 6 套主题的视觉验证
- 不改变现有的 Tailwind CSS 配置

---

## 优先级

| 优先级 | 需求 | 理由 |
|--------|------|------|
| P0 | 统一间距和布局 | 基础视觉一致性，影响最大 |
| P0 | 主题闪烁修复 | 用户体验问题，首次加载即可见 |
| P1 | 微交互优化 | 提升质感，但不影响功能 |
| P1 | 移动端交互 | 移动端用户无法使用核心功能 |

---

## 不在范围内

- 新增主题或修改现有主题颜色
- 添加新的动画效果（如 Framer Motion）
- 重构组件架构
- 修改 Tailwind 配置
