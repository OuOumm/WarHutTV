## 播放器：增加「下一集」按钮 + 自动连播
- 需求：播放器内增加下一集按钮，且一集放完自动跳下一集。
- 改动文件与要点：
  - `frontend/src/pages/play/useEpisodeNav.ts`：新增 `handleNextEpisode()`，按 `stateRef.current.currentEpisode` 在 `episodes` 的索引找下一集并调 `handleEpisodeClick`，返回是否有下一集（boolean）。
  - `frontend/src/components/Player.tsx`：新增 props `onNext / onEnded / hasNext`（用 ref 持有最新值，避免重建播放器）。创建时 `art.controls.add({ name:'nextEpisode', position:'right', html:'下一集 ▶', tooltip:'下一集', click:()=>onNextRef.current?.() })` 注册控制条右侧「下一集」控件；`art.on('video:ended', ()=>onEndedRef.current?.())` 实现自动连播。hasNext 变化时通过 `nextControlRef.current.style`（opacity/pointerEvents/cursor）切换禁用态。控件元素在 cleanup 时置 null。
  - `frontend/src/pages/play/PlayerViewport.tsx`：Props 增加 `onNext/onEnded/hasNext` 并透传给 `<Player>`。
  - `frontend/src/pages/play/usePlayController.ts`：暴露 `handleNextEpisode` 与 `hasNextEpisode`（useMemo 计算索引）。
  - `frontend/src/pages/Play.tsx`：`<PlayerViewport>` 接 `onNext/onEnded={controller.handleNextEpisode}`、`hasNext={controller.hasNextEpisode}`；标题旁加可键盘访问的「下一集 ▶」按钮（hasNextEpisode 为真时显示）。
- 验证：`tsc -b` 0 error、`vitest` 29/29、`vite build` 成功（仅既有 `INEFFECTIVE_DYNAMIC_IMPORT` 警告，与本改动无关）。

## 继续观看：跳转对应集数 + 对应进度（deep-link 续播）
- 需求：从首页「继续观看」卡片点进去，直接跳到上次最后播放的集数 + 进度。
- 链路与真相来源：`WatchHistory.playback_key = site:vodId:集名`，`updateProgressByContext` 会把最后观看集写进 playback_key；但 `episode` 字段对旧记录不可靠（停留在首集名），故续播集名从 playback_key 解析（slice(2).join(':') 兼容集名含冒号）。
- 改动：
  - `src/components/VideoCard.tsx`（VOD 分支）：从 `playback_key` 解析真实最后观看集名 + `progress`，拼到 `/play/:site/:id?ep=<集名>&t=<进度秒>`（URLSearchParams 自动编码）。仅当存在 resume 数据才加参数（搜索/收藏卡片无 progress→默认行为不变）。
  - `src/pages/play/playContext.ts`：`PlayControllerDeps` 加 `initialEpisode?` / `initialTime?` / `episodesPerPage`。
  - `src/pages/play/usePlayController.ts`：`useSearchParams()` 读 `ep`/`t`，算 `initialEpisode`/`initialTime`，经 deps 下传。
  - `src/pages/play/playUtils.ts`：新增 `resolveResumeEpisode(episodes, initialEpisode?)`（有 initialEpisode 按名精确匹配，否则回退 epList[0]，保旧行为）、`episodePageIndex(episodes, target, perPage)`、`applyResumeProgress(setCurrentTime, toast, site, vodId, episode, initialTime?)`（优先 initialTime，否则读该集历史进度；仅 >0 时 setCurrentTime + toast）。
  - `src/pages/play/useDetailLoader.ts`：detail 加载后 `resolveResumeEpisode` 定 `currentEpisode` + `episodePageIndex` 定可见页；删掉原先只对首集读进度的逻辑（进度恢复统一交给 optimize 路径，避免重复 toast）。`historyStore.add` 的 episode 改为 `targetEp?.name`。
  - `src/pages/play/useSourceSearch.ts`：`playFallback` / 单源路径 / `applyPicked` / catch 四处原先 `applyHistoryProgress(...,'')` 全改为 `applyResumeProgress(... targetEp, deps.initialTime)`，并 dispatch `currentEpisode: targetEp` + `episodePage`（关键：startOptimize 应用源时 `applySource` 会把 currentEpisode 重置回 epList[0]，必须在其后 patch 回 targetEp，否则跳错集）。
  - `src/store/history.ts`：`updateProgressByContext` 更新进度时同步写 `episode` 字段（existing 与 fallback 分支），保证 record.episode 未来可靠。
- Player 时机：Player 仅在挂载/`art.switch` 换源时 seek `seekTimeRef`（=最新 currentTime），故只要换源后 currentEpisode+currentTime 定位对，即自动续播到正确集+进度。
- 验证：`tsc --noEmit` 0、`vitest` 29/29、`vite build` 成功、`eslint` 0 error（仅 `optimizeStarted.current=true` 的 react-hooks/immutability 警告，为改动前既有，与本功能无关）。
