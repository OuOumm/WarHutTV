# 2026-07-08

## 修复：移动端卡片播放按钮遮挡标题
- 文件：`frontend/src/components/VideoCard.tsx`
- 问题：桌面端 hover 用的 `PlayOverlay` 播放按钮在移动端作为覆盖层存在（`z-[3]`），卡片标题容器仅 `z-[1]`，窄屏交互时可能压住标题。
- 修复：
  1. `PlayOverlay` 容器改为 `hidden sm:flex` —— 移动端 (<640px) `display:none` 彻底不渲染；桌面端 hover 行为不变。
  2. 标题容器（Bottom info）层级由 `z-[1]` 提到 `z-[5]`，为卡片内最高，确保标题永不被任何元素覆盖。
- 验证：`npm run build` 通过；built CSS 确认 `.hidden{display:none}`、`@media(width>=40rem){.sm\:flex{display:flex}}`、`z-index:5` 均存在。
- 说明：PlayOverlay 仅为桌面 hover 装饰，卡片本身是 `<Link>` 点击即跳转播放页，移动端移除播放按钮不影响功能。
